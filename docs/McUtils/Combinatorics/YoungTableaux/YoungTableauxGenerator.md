**LLM Docstring**

Count standard Young tableaux over one or more partitions of the instance base.

If `partitions` is omitted, integer partitions of `self.base` are generated. A single partition is wrapped, while nested partition groups are flattened before summing the hook-length counts.
  - `partitions`: `array-like | None`
    > a single partition, a flat collection of partitions, or nested partition groups
  - `partition_opts`: `Any`
    > options forwarded to `IntegerPartitioner.partitions` when partitions are generated
  - `:returns`: `int`
    > total number of standard tableaux for all selected partitions

**LLM Docstring**

Generate standard Young tableaux for selected partitions.

The method normalizes single, flat, and nested partition inputs, then delegates each partition to `standard_partition_tableaux`. When requested, it returns the normalized partitions alongside the generated row-array representation.
  - `partitions`: `array-like | None`
    > a single partition, flat partition collection, or nested partition groups; defaults to partitions of `self.base`
  - `symbols`: `array-like | None`
    > optional values used to replace the default integer labels by indexed lookup
  - `brute_force`: `bool`
    > use permutation filtering instead of the recursive construction
  - `return_partitions`: `bool`
    > return the normalized partition specification with the tableaux
  - `partition_opts`: `Any`
    > options forwarded to `IntegerPartitioner.partitions`
  - `:returns`: `list[numpy.ndarray] | tuple[list[numpy.ndarray], array-like]`
    > generated tableau arrays, optionally paired with their partitions

**LLM Docstring**

Enumerate tableaux for a partition by brute-force permutation filtering.

Rows are represented as separate arrays whose first axis indexes candidate tableaux. Candidates are retained only when every row and every transposed column is nondecreasing. With `unique_perms=True`, row-membership permutations are used to derive rank positions; otherwise all permutation indices of `sum(partition)` are considered.
  - `partition`: `sequence[int]`
    > row lengths of the Young diagram
  - `unique_perms`: `bool`
    > derive candidates from unique permutations of repeated row labels
  - `concatenate`: `bool`
    > concatenate row arrays along the column axis before returning
  - `:returns`: `list[numpy.ndarray] | numpy.ndarray`
    > filtered tableau rows or their concatenation

Fill tableau frame shapes from segmented symbol selections.

For each entry in `segment_lists`, the function allocates one integer row per partition row and copies successive slices from each segment according to the counts stored in `frame`. Segment offsets are tracked independently for each segment index.

    > iterable of segment collections supplying the values
  - `:returns`: `list[list[numpy.ndarray]]`
    > list of populated subframes, each represented as a list of row arrays

**LLM Docstring**

Compute hook lengths for every cell of a Young diagram.

Cells are traversed row by row. Each hook length is the number of cells from the current position through the end of its row plus one for every lower row extending into the same column.
  - `partition`: `sequence[int]`
    > row lengths of the Young diagram
  - `:returns`: `list[int]`
    > hook lengths in row-major order

Count standard tableaux using the hook-length formula.

The function computes `n! / product(hooks)` as a product of elementwise ratios, rounds the floating-point result, and converts it to an integer NumPy scalar.

  - `:returns`: `numpy.integer`

**LLM Docstring**

Generate all standard tableaux for a single partition.

The method selects either brute-force permutation filtering or the recursive `_sst_2` implementation. If `symbols` is provided, each integer index in every row array is replaced by the corresponding symbol.
  - `partition`: `sequence[int]`
    > row lengths of the Young diagram
  - `cache`: `dict | None`
    > optional recursive tableau cache
  - `subset_cache`: `dict | None`
    > optional unique-subset cache
  - `symbols`: `array-like | None`
    > optional label array indexed by generated tableau entries
  - `brute_force`: `bool`
    > use `standard_partition_tableaux_bf` instead of `_sst_2`
  - `:returns`: `list[numpy.ndarray]`
    > list of row arrays with the tableau index along axis 0
